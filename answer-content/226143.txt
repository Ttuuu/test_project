<p>This is the key insight.  A chunk of code is likely to be <em>read</em> more times than it was <em>written</em>.  So write your code to be read.  Use comments, docstrings, whitespace, type hints, etc. to make it easier for someone unfamiliar with the code (including your future self) to read and understand it.</p>
