<p>Too me the biggest thing to look at in your code is the complexity.  Right now it's O(nÂ²).  This is very inefficient and can be greatly improved upon, by taking a step back and changing the <code>headers</code> to a <code>HashMap&lt;String, Integer&gt;</code>.  Now, since lookups for <code>HashMap</code> is O(1) you only need to count the loop through the input array, which will give a complexity of O(n).  The code could look something like this:</p>

<pre><code>HashMap&lt;String, Integer&gt; headers = new HashMap&lt;&gt;();

public int[] findIndices(String... input) throws IllegalArgumentException {
    int headerSize = headers.size();
    if (input.length &gt; headerSize) {
        throw new IllegalArgumentException("Amount of searched for headers, " + input.length
                + ", exceeded the actual amount of headers: " + headerSize);
    }
    int[] indices = new int[input.length];
    int index = 0;
    for (String s : input) {
        if (headers.containsKey(s)) {
            indices[index++] = headers.get(s);
        } else {
            throw new IllegalArgumentException("One or more of input arguments could not be found in the headers.");
        }
    }
    return indices;
}
