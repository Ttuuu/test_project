<p>Both examples, A &amp; B are not very nice.</p>

<p>There is a lot of business logic in the controller. Code in controllers is not re-usable. </p>

<p>You can put the validation code into the model (as a method) to make it more re-usable. </p>

<p>In general:
Controllers should be small and contain little/less code
Models should contain all business logic including validation rules</p>
