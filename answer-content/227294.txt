<p>You probably are familiar with <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date" rel="nofollow noreferrer"><code>&lt;input type="date"&gt;</code></a> but maybe you aren't satisfied by the <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date#Browser_compatibility" rel="nofollow noreferrer">browser compatibility</a> or wanted to try your hand at  <a href="/questions/tagged/reinventing-the-wheel" class="post-tag" title="show questions tagged 'reinventing-the-wheel'" rel="tag">reinventing-the-wheel</a>.</p>

<p>Instead of creating numerous HTML elements with the Javascript Logic, have you considered using a <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/template" rel="nofollow noreferrer"><code>&lt;template&gt;</code></a> with <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/slot#" rel="nofollow noreferrer"><code>&lt;slot&gt;</code></a> tags and replacing values as needed.</p>

<p>Also, I see many variables declared with <code>let</code> but only one with <code>const</code> (i.e. <code>newMonth</code>). It is wise to use <code>const</code> as a default and then when it is determined that re-assignment is necessary switch to <code>let</code>.</p>

<p>Instead of assigning a reference to <code>this</code> in <code>vm</code> use arrow functions (because there would be no separate <code>this</code> context) or else set the context of those anonymous functions to <code>this</code> using <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/bind" rel="nofollow noreferrer"><code>Function.bind()</code></a>.</p>
