<p>Except the formatting, your code does not looks ugly. It just looks procedural.</p>

<p>Try to create classes for each roles. One that read and write to the console. One to compute the loan. And another to control the flow of your program. </p>

<p>As said by Martin Fowler, <em>"Any fool can write code that a computer can understand. Good programmers write code that humans can understand."</em></p>

<p>"understand" is not only about being able to figure what your code does. But also about maintainability and evolvability.</p>
